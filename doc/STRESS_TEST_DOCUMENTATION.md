## 1. 概述

本压测方案旨在全面评估 HCC（High Consistency Cache）缓存系统的性能表现，包括：

- **单机读写并发容量**：确定单节点能够支持的最大并发读写量
- **系统瓶颈定位**：识别性能瓶颈所在层（本地缓存/Redis/网络）
- **稳定性验证**：验证系统在极端场景下的稳定性和容错能力
- **容量规划**：为生产环境部署提供容量参考数据

## 4. 测试类详解

### 4.1 本地缓存层压测

**文件路径**：`LocalCacheStressTest.java`

**测试目标**：评估Caffeine本地缓存的极限性能

#### 4.1.1 写入极限测试

| 参数 | 值 |
|------|-----|
| 线程数 | 10, 50, 100, 200 |
| 每线程操作数 | 10,000 |
| 数据大小 | 约50字节/条 |

**测试逻辑**：
- 多线程并发写入不同key
- 测量总吞吐量和QPS

**预期结果**：
- 写入QPS：≥ 50,000 ops/sec（10线程）
- 写入QPS：≥ 100,000 ops/sec（100线程）

#### 4.1.2 读取极限测试

| 参数 | 值 |
|------|-----|
| 预热数据量 | 50,000条 |
| 线程数 | 10, 50, 100, 200, 500 |
| 每线程操作数 | 20,000 |

**测试逻辑**：
- 随机读取预热数据
- 统计命中率和QPS

**预期结果**：
- 读取QPS：≥ 200,000 ops/sec（100线程）
- 命中率：≥ 99%

#### 4.1.3 读写混合测试

| 参数 | 值 |
|------|-----|
| 预热数据量 | 20,000条 |
| 线程数 | 50, 100, 200 |
| 读写比例 | 9:1, 7:3, 5:5 |

**测试逻辑**：
- 根据比例随机执行读或写操作
- 测量混合场景下的吞吐量

**预期结果**：
- 9:1比例下QPS：≥ 150,000 ops/sec

#### 4.1.4 驱逐性能测试

| 参数 | 值 |
|------|-----|
| 缓存最大容量 | 1,000条 |
| 插入总数 | 50,000条 |
| 线程数 | 20, 50, 100 |

**测试逻辑**：
- 插入超过容量的数据
- 测量LRU驱逐效率

**预期结果**：
- 最终缓存大小：≈ 1,000条
- 插入QPS：≥ 30,000 ops/sec

## 6. 预期结果分析

### 6.1 性能基准

| 场景 | 预期QPS | 预期延迟 |
|------|---------|---------|
| 纯本地读 | ≥ 200,000 | < 1ms |
| 纯本地写 | ≥ 100,000 | < 1ms |
| Redis标记操作 | ≥ 10,000 | < 1ms |
| 完整链路读 | ≥ 50,000 | < 50ms |
| 完整链路写 | ≥ 10,000 | < 100ms |

### 6.2 容量估算

基于测试结果，可按以下公式估算生产容量：

```
生产QPS = 单节点QPS × 节点数 × 安全系数(0.7)

推荐配置：
- 8核16GB节点：单节点QPS ≈ 50,000
- 目标100,000 QPS：需要3个节点
```

| 指标 | 说明 |
